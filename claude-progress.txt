🔧 TECHNOLOGY STACK & ARCHITECTURAL IRON LAWS
================================================================================

Backend Framework:
  - FastAPI (uvicorn on port 8002)
  - LangGraph for workflow orchestration
  - Pydantic v2 for data validation
  - AsyncSqliteSaver for state persistence

Frontend Stack:
  - Vue 3 (CDN-based, no build step)
  - Tailwind CSS for styling
  - Axios for HTTP requests
  - Native EventSource for SSE streaming

Agent Workflow Nodes:
  1. Intent Node       : backend/app/agent/nodes/intent_node.py
  2. Retrieval Node    : backend/app/agent/nodes/retrieval_node.py
  3. Planner Node      : backend/app/agent/nodes/planner_node.py
  4. Executor Node     : backend/app/agent/nodes/executor_node.py
  5. Analyzer Node     : backend/app/agent/nodes/analyzer_node.py

State Management:
  - AgentState (backend/app/agent/state.py) - TypedDict with:
    * messages, query, extracted_filters
    * plan, current_step, data_context
    * requires_approval

Graph Definition:
  - backend/app/agent/graph.py
  - Conditional edges for approval gates and execution loops

API Endpoints:
  - backend/app/api/v1/chat.py (SSE streaming endpoint)
  - backend/app/api/v1/api_management.py
  - backend/app/api/v1/auth.py, user_management.py, etc.

🚫 ARCHITECTURAL IRON LAWS (DO NOT VIOLATE):
  1. All state MUST pass through AgentState - no global variables for workflow data
  2. All user-facing events MUST emit via SSE - no silent failures
  3. All API calls MUST go through RBAC permission checks
  4. All LLM outputs MUST be validated against Pydantic schemas
  5. Frontend uses Vue 3 + Tailwind - DO NOT introduce React/Angular/other frameworks
  6. Backend uses FastAPI + LangGraph - DO NOT introduce Flask/Django
  7. DO NOT add new npm dependencies - frontend is CDN-based
  8. DO NOT replace existing libraries (axios, marked, chart.js) with alternatives
