===========================================
二、技术栈与规范铁律
===========================================

【后端技术栈】
- 框架：FastAPI + Uvicorn
- 推理引擎：LangGraph (Plan-and-Execute 模式)
- 状态管理：SQLite Checkpointer (持久化 AgentState)
- LLM 调用：OpenAI SDK (兼容多种 LLM API)
- 数据处理：Pandas + NumPy
- 异步支持：aiohttp + httpx + aiomysql + asyncpg

【前端技术栈】
- 纯静态 HTML/CSS/JavaScript
- SSE (Server-Sent Events) 流式通信

【核心架构】
- LangGraph 状态机：Intent → Retrieval → Planner → Executor → Analyzer
- 数据隔离：所有 API 查询结果存入 data_context 字典，避免 Prompt 污染
- 人工审批网关：interrupt_before=['Executor'] 实现审批挂起
- 权限控制：RBAC 基于用户角色和 API 授权

【禁止事项】🚫
- 禁止引入同类替代框架（如 LangChain、Haystack）
- 禁止修改 AgentState 顶层结构
- 禁止破坏 LangGraph 主流程
- 禁止在 Prompt 中拼接大量数据（必须使用 data_context）
- 禁止擅自删除审批模块代码（仅调整触发条件）

2. 修改 main.py：
   - 在 lifespan 启动阶段添加 API 向量索引构建
   - 遍历所有 API 并调用 build_index_for_api()

3. 修改 api_retrieval_service.py：
   - 在 _enrich_with_full_metadata() 中规范化 endpoint 结构
   - 将 params_mapping 复制到 params 字段
   - 确保 Planner 能正确读取参数信息

===========================================

一、任务目标
===========================================

【功能ID】F-06
【优先级】6
【类型】bugfix
【描述】规范化股票API返回结果，确保Analyzer能提取有效数据行

【问题背景】
Alpha Vantage股票API返回的是嵌套的时间序列数据：
```json
{
  "Meta Data": {...},
  "Time Series (Daily)": {
    "2024-03-27": {"1. open": "...", "2. high": "...", ...},
    "2024-03-26": {...}
  }
}
```

当前代码直接将原始JSON存入data_context，导致Analyzer的_extract_all_data()
只能提取到1个dict对象，而不是多行交易日数据，出现"Stored result成功但Extracted 0 total rows"问题。

===========================================
二、实现方案
===========================================

【核心修改】
1. 在api_fetch_tool.py中添加_normalize_stock_data()方法
2. 在返回数据前对股票API进行特殊处理
3. 更新analyzer_node.py的_extract_all_data()识别规范化格式

【规范化数据结构】
```python
{
  "rows": [
    {
      "date": "2026-03-26",
      "open": 252.115,
      "high": 257.0,
      "low": 250.77,
      "close": 252.89,
      "volume": 41486694.0,
      "pct_change": 0.31  # 自动计算涨跌幅
    },
    ...
  ],
  "row_count": 100,
  "source": "alpha_vantage",
  "symbol": "AAPL",
```
