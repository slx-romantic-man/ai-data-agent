===========================================
五、工作纪律 (DISCIPLINE)
===========================================

🚫 禁止行为：
- 禁止一次性实现多个功能
- 禁止仅凭代码存在就判断项目完成
- 禁止跳过测试步骤直接标记 passes=true
- 禁止修改已完成功能的代码 (除非发现 bug)

✅ 必须遵守：
- 每次会话只做一个功能 (按 priority 选择最高优先级的未完成功能)
- 完成后必须执行 feature_list.json 中的所有 steps 进行验证
- 验证通过后才能将 passes 字段改为 true
- 每次完成功能后必须更新本文件并提交 Git
- feature_list.json 是唯一真相来源 (Single Source of Truth)

===========================================
六、失败模式警告 (FAILURE MODES)
===========================================

⚠️ 常见失败模式及应对：

1. "我看代码已经存在了，应该是完成了"
   → 错误！必须以 feature_list.json 为准，执行测试步骤验证

2. "我一次性实现了 3 个功能，效率更高"
   → 错误！这会导致代码质量下降和难以回滚

3. "测试步骤太麻烦，我直接看代码逻辑正确就行"
   → 错误！必须端到端验证，代码正确不等于功能可用

4. "我发现之前的代码有问题，顺便重构一下"
   → 错误！除非是阻塞性 bug，否则不要修改已完成的功能
