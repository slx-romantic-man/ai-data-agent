================================================================================
  二、技术栈与规范铁律 (🚫 禁止擅自引入同类替代品)
================================================================================

  [后端]
  - 语言:       Python 3.8+
  - 框架:       FastAPI (uvicorn)
  - AI 编排:    LangGraph (StateGraph + AsyncSqliteSaver)
  - LLM 客户端: 自研 BaseLLMClient (OpenAI/Aliyun/Anthropic compatible)
  - HTTP 客户端: httpx (异步)
  - 数据库:     SQLAlchemy 2.0 (async) + aiosqlite
  - 向量搜索:   Qdrant (qdrant-client) + sentence-transformers
  - 数据处理:   Pandas + NumPy
  - 缓存:       自研 LLMResponseCache (app/utils/llm_cache.py)
  - 导出:       openpyxl (Excel)

  [前端]
  - 类型:       纯静态 HTML/CSS/JS (无框架, 无构建步骤)
  - 入口:       frontend/index.html
  - 通信:       SSE (Server-Sent Events) + REST API

  [基础设施]
  - 日志:       自研 app/utils/logger.py
  - 配置:       pydantic-settings (.env 驱动)
  - 部署:       Docker + docker-compose + nginx
  - 状态持久化: SQLite (WAL 模式)

  🚫 禁止引入: Flask, Django, LangChain (非 LangGraph), axios,
               React, Vue, Angular, Webpack, Vite, TailwindCSS,
               或任何与上述栈功能重叠的替代品。

================================================================================
  四、核心工作流 (LangGraph 节点链)
================================================================================

  START → Retrieval → IntentPlanner → Executor (循环) → Analyzer → END

  性能瓶颈:
  - IntentPlanner LLM 调用: ~42s (prompt 过长, max_tokens=4096)
  - Analyzer LLM 调用:      ~85s (7段式报告, prompt 臃肿)
  - 总计 LLM 占比:          96%

================================================================================
  六、工作纪律 (🚨 后续每个 AI Agent 必须遵守)
================================================================================

  1. 每次会话只做一个功能
     - 打开 feature_list.json
     - 找到 priority 最小且 passes=false 的功能
     - 只做那一个

  2. 完成后更新状态
     - feature_list.json 中该功能的 passes 改为 true
     - 在本文件 "七、进度日志" 追加一条记录

  3. Git 提交
     - 提交格式: feat(perf): F-XX 功能简述
     - 例如: feat(perf): F-01 skip analyzer for simple queries

  4. 失败模式警告
     🚫 禁止一次性实现多个功能
     🚫 禁止仅凭代码存在就判断功能完成，必须以 feature_list.json 为唯一真相
     🚫 禁止引入与现有技术栈重叠的新依赖
     🚫 禁止修改不在当前功能 files_to_modify 列表中的文件（除非是必要联动）
