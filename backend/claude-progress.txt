## 任务目标

为股票、天气、IP 三个外部 API 各设计 3 条不同表述的自然语言问题，验证 Agent 的鲁棒性和最终回答质量。

## 问题诊断与修复

### 问题1: 系统阻塞问题（上一任遗留）

**症状**：
- 单个请求超时（60秒+）
- 连续请求完全阻塞
- 日志显示请求卡在 Analyzer 节点调用 LLM

**根本原因**：
1. LLM 提供商不稳定：MiniMax (api.minimaxi.com) 响应超时
2. SQLite checkpoint 数据库并发锁定：多个请求同时访问 checkpoints.db 导致 "database is locked"
3. MySQL 连接池问题：使用 NullPool 导致连接耗尽

**解决方案**：

1. **切换 LLM 提供商**（.env）
   - 从 MiniMax 切换到 aiznt.com
   - 使用 claude-sonnet-4-5-20250929 模型
   - 验证：单个请求从超时变为 70 秒成功响应

2. **修复 SQLite 并发问题**（backend/app/agent/graph.py）
   - 启用 WAL 模式：`PRAGMA journal_mode=WAL`
   - 增加超时时间：`timeout=30.0`
   - 设置 busy_timeout：`PRAGMA busy_timeout=30000`
   - 允许多线程访问：`check_same_thread=False`

3. **系统重启清理锁文件**
   - Qdrant 锁文件无法通过代码释放
   - 需要系统重启才能彻底清理

## 技术要点

1. **LLM 提供商稳定性至关重要**
   - MiniMax 不稳定，导致整个系统阻塞
   - aiznt.com 稳定，响应时间约 60-70 秒

2. **SQLite 并发处理**
   - WAL 模式允许并发读写
   - busy_timeout 防止立即失败
   - 适合中低并发场景

3. **测试脚本超时设置**
   - 原 60 秒超时不足
   - 建议 120 秒以上
