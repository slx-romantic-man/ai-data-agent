# CHANGELOG

## [v4.0.0] - 2024-03-26

### 🎉 重大架构升级：从 ReAct 到 LangGraph

本版本完成了从 v3 ReAct 循环架构到 v4 LangGraph Plan-and-Execute 架构的全面升级。

### ✨ 新增特性

#### 核心架构 (F-01 ~ F-07)
- **F-01**: 引入 LangGraph 库并定义 AgentState 数据结构
- **F-02**: 实现 Intent Clarification Node（意图澄清节点）
- **F-03**: 实现 API Retrieval Node（工具检索节点）
- **F-04**: 实现 Planner Node（全局规划师节点）
- **F-05**: 实现 Executor Node（无脑执行机节点）
- **F-06**: 实现 Analyzer Node（终局分析节点）
- **F-07**: 构建 LangGraph 工作流图并配置边与路由

#### 持久化与集成 (F-08 ~ F-10)
- **F-08**: 接入 AsyncSqliteSaver 实现状态持久化
  - 支持多轮对话上下文保存
  - `data_context` 持久化避免重复查询
- **F-09**: 更新 API 层接入新的 LangGraph 工作流
  - 保持 SSE 流式输出格式不变
  - 兼容前端现有交互逻辑
- **F-10**: 实现人工审批网关前后端交互
  - 基于用户角色的条件审批机制
  - 管理员直接执行，普通用户需审批
  - 新增 `/api/v1/chat/approve` 和 `/api/v1/chat/reject` 端点

#### 端到端测试 (F-11 ~ F-14)
- **F-11**: 简单查询场景测试（修复表结构查询 bug）
- **F-12**: 复杂归因分析场景测试
- **F-13**: 多轮下钻场景测试
- **F-14**: 人工审批网关场景测试

#### Python 代码执行能力 (F-16 ~ F-19)
- **F-16**: 实现 Python 代码执行工具（PythonExecTool）
  - 隔离环境执行代码（RestrictedPython）
  - 支持 Pandas、NumPy 等数据处理库
- **F-17**: 更新 Planner 支持 python_exec 工具类型
  - 在 Prompt 中添加 Python 代码执行工具描述
  - 支持数据注入和变量引用
- **F-18**: 更新 Executor 路由支持 Python 代码执行
  - 添加 `python_exec` 路由分支
  - 实现数据注入逻辑
- **F-19**: 端到端测试：数学运算场景
  - 场景1：简单计算（增长率）
  - 场景2：基于 API 数据计算（平均值）
  - 场景3：多步骤依赖（利润率）

### 🔧 改进

#### 架构优化
- **数据隔离存储**: 所有查询结果存入 `data_context` 字典，避免 Prompt 污染
- **确定性流程**: Plan-and-Execute 模式替代 ReAct 循环，避免目标丢失
- **状态持久化**: 基于 SQLite 的会话状态管理，支持多轮对话恢复

#### 审批机制优化
- 从"所有用户调用高危 API 触发审批"改为"基于用户角色的条件审批"
- 管理员（role=admin）直接执行，无需审批
- 普通用户（role=user）调用 API 时需要审批

### 🐛 Bug 修复

- **F-11**: 修复表结构查询 bug
  - 问题：查询表结构时返回空结果
  - 原因：SQL 查询条件错误
  - 解决：更新 SQL 查询逻辑

### 📝 文档更新

- **F-15**: 更新项目文档反映 v4 架构变更
  - 更新 README.md 添加 LangGraph 架构说明
  - 更新 API 文档添加审批接口说明
  - 添加 v3 vs v4 对比表格

### 🔄 需求变更记录

#### [2024-03-24] F-10 需求变更
- **原需求**: 所有用户调用高危 API 时触发审批
- **新需求**: 基于用户角色的条件审批
  - 管理员（role=admin）：直接执行，无需审批
  - 普通用户（role=user）：调用 API 时需要审批

#### [2024-03-24] 新增 Python 代码执行能力
- **背景**: 数学运算场景需要严谨的计算能力
- **需求**: 支持在隔离环境中执行 Python 代码
- **影响**: 新增 F-16 ~ F-19 四个功能点

### 📊 技术栈更新

#### 新增依赖
- `langgraph>=0.2.0` - LangGraph 状态机框架
- `langgraph-checkpoint-sqlite>=1.0.0` - SQLite 状态持久化
- `aiosqlite>=0.17.0` - 异步 SQLite 客户端
